FLUJO DE PROCESO API:

1
Solicitud de pago vía API
La entidad bancaria envía solicitud con datos del pago hacia el sistema
2
Validación de datos
Sistema valida recaudador, importe, forma de pago, fecha, cliclave, etc.
3
Consulta del contribuyente
✓ Consulta exitosa → continúa procesamiento
✗ Consulta fallida → devuelve error y termina
4
Inserción de pago
✓ Inserción exitosa → confirma y registra pago
✗ Inserción fallida → devuelve error para corrección
↓

B PAGO VÍA ARCHIVO / PLANTILLA

REQUISITOS DEL ARCHIVO:
  • Archivo o plantilla con información de pagos bancarios
  • Cada institución puede tener su plantilla específica
  • Campos obligatorios: fecha, importe, forma de pago
  • Formato: archivo de texto, preferentemente una sola línea
  • Delimitadores aceptados: | o @ (alternativa: tabulador)
  • Nota: código de barras es solo para lectura, no para inserción
MODALIDADES DE PROCESAMIENTO:
  • Procesamiento Manual: El operador envía el archivo y el sistema lo valida paso a paso, permitiendo correcciones antes de confirmar
  • Procesamiento Automático: El archivo se procesa automáticamente según un horario programado sin intervención del operador

FLUJO DE PROCESO ARCHIVO/PLANTILLA:

1
Recepción del archivo
Sistema ATL recibe archivo o plantilla de la entidad bancaria
2
Validación de estructura
✓ Archivo correcto → procede con registro
✗ Archivo incorrecto → genera reporte de error
3
Inserción de pagos
Sistema procesa e inserta todos los pagos del archivo en ATL
4
Confirmación de recepción
Sistema genera reporte de estado de cada pago (exitoso o con errores)

    C PAGO VÍA CAJAS

    DESCRIPCIÓN DEL PROCESO:
    • Pago realizado en las cajas de atención al público
    • El ciudadano se presenta en persona para realizar el pago
    • Operador de caja registra el pago en el sistema ATL
    • Se puede realizar la carga manual al abrir una Sesión de Recaudación Externa
    SESIÓN DE RECAUDACIÓN EXTERNA:
    • Simula la apertura de una sesión de caja sin fondo inicial
    • Permite registrar pagos recibidos de forma manual
    • Cada pago ingresado se acumula en la sesión
    • Al finalizar, se genera un cierre con el total de recaudación
    • Ideal para operadores móviles o puntos de recaudación temporales
    DATOS REQUERIDOS EN CAJA:
    • Identificación del usuario (cuenta, RFC, referencia)
    • Importe a pagar
    • Forma de pago (efectivo, tarjeta, cheque, etc.)
    • Fecha y hora de la transacción
    • Referencia de comprobante o recibo

    FLUJO DE PROCESO CAJAS:

    1
    Atención en caja
    Ciudadano se presenta en la caja con intención de realizar un pago
    2
    Solicitud de información
    Operador de caja solicita identificación y referencia del adeudo
    3
    Registro en sistema ATL
    Operador captura datos: importe, forma de pago, fecha, referencia
    ✓ Registro exitoso → sistema confirma pago
    ✗ Error en registro → solicita corrección de datos
    4
    Emisión de comprobante
    Sistema genera y emite recibo o comprobante de pago al ciudadano
    5
    Registro finalizado
    Pago registrado en sistema ATL y disponible en historial del usuario

    PROCESO DE TIMBRADO Y CONCILIACIÓN

    📋 TIMBRADO DE COMPROBANTES FISCALES

    PROCESO DE TIMBRADO DIARIO:
    • Timbrado Individual: Se genera diariamente un Comprobante Fiscal Digital por Internet (CFDI) individual para cada pago registrado
    • Emisión Automática: El sistema genera los timbres de forma automática al finalizar el día
    • Validación SAT: Cada comprobante es validado y certificado ante el Servicio de Administración Tributaria (SAT)
    • Entrega al Contribuyente: Los CFDIs individuales están disponibles para descarga inmediata por el usuario
    TIMBRADO GLOBAL MENSUAL:
    • Consolidación Mensual: Al finalizar el mes se genera un CFDI Global que agrupa todas las transacciones del periodo
    • Resumen de Operaciones: Incluye el total de pagos procesados por todas las vías (API, Archivo, Cajas)
    • Archivo para Instituciones: El timbrado global se entrega a las instituciones bancarias y recaudadoras
    • Cumplimiento Fiscal: Garantiza el cumplimiento de las obligaciones fiscales del organismo operador

    CALENDARIO DE TIMBRADO:

    FRECUENCIA TIPO DE TIMBRADO DESTINATARIO GENERACIÓN
    Diario CFDI Individual Contribuyente/Usuario Automática - Fin del día
    Mensual CFDI Global Instituciones Bancarias Automática - Último día del mes
    Bajo Demanda Re-impresión CFDI Contribuyente/Usuario Manual - Portal web/APP

    ⚖️ PROCESO DE CONCILIACIÓN BANCARIA

    CONCILIACIÓN MANUAL:
    • Validación Completa: Una vez confirmados y validados todos los pagos del día, se procede con la conciliación manual
    • Comparación de Registros: Se comparan los registros del sistema ATL contra los reportes bancarios y de recaudadores
    • Identificación de Diferencias: Se detectan y documentan posibles discrepancias o diferencias
    • Resolución de Inconsistencias: Se investigan y resuelven las diferencias encontradas antes del cierre
    • Aprobación Final: El área de finanzas aprueba la conciliación diaria

    FLUJO DE CONCILIACIÓN DIARIA:

    1
    Recopilación de información
    Se obtienen reportes del sistema ATL y estados de cuenta bancarios del día
    2
    Validación de pagos confirmados
    Se verifica que todos los pagos registrados estén confirmados y validados
    3
    Comparación y análisis
    Se realiza la conciliación manual comparando registros internos vs. bancarios
    ✓ Conciliación exitosa → procede al cierre
    ✗ Diferencias detectadas → investigación y corrección
    4
    Generación de reporte de conciliación
    Se documenta el resultado de la conciliación con todos los detalles
    5
    Notificación por correo electrónico
    Se envían correos a las áreas involucradas indicando qué días están conciliados
    NOTIFICACIONES POR CORREO ELECTRÓNICO:
    • Destinatarios: Áreas de Finanzas, Contabilidad, Dirección y Sistemas
    • Contenido del Correo: Fecha conciliada, total de transacciones, importe total, número de instituciones
    • Periodicidad: Diaria - Al completar la conciliación del día
    • Formato: Reporte adjunto en PDF con detalle de la conciliación
    • Estatus: Se indica claramente qué días están 100% conciliados y cuáles tienen pendientes
